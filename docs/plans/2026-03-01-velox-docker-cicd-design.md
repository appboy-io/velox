# Velox Docker & CI/CD Design

## Goal

Make Velox easy to adopt in CI/CD pipelines (GitHub Actions, CircleCI, Bitbucket Pipelines) by shipping a Docker image with Gatling + Velox pre-installed, and providing workflow scaffolding.

## Distribution

- **Primary (CI/CD):** Docker image at `ghcr.io/appboy-io/velox`
- **Secondary (local dev):** `pip install velox` for users who have Gatling installed locally

## Docker Image

- **Base:** Eclipse Temurin 21 JRE (slim) + Python 3.10+
- **Contents:** Gatling pre-installed, Velox pip-installed, `GATLING_HOME` set
- **Entrypoint:** `velox` CLI — `docker run ghcr.io/appboy-io/velox run /tests/perf.yaml`
- **Test YAML:** Mounted via volume (`-v ./tests:/tests`)
- **Env vars:** `TARGET_URL` and any other variables passed through with `-e`
- **Exit code:** Non-zero when thresholds are breached (fails the pipeline)

## Tagged Release Workflow

A GitHub Actions workflow in the Velox repo:

- **Trigger:** Push tags matching `v*`
- **Steps:** Checkout, log in to GHCR, build Docker image, tag as version + `latest`, push
- **Output:** `ghcr.io/appboy-io/velox:v0.1.0` and `ghcr.io/appboy-io/velox:latest`

## `velox init --ci` Command

Extends the existing `init` command:

- `velox init --ci github` — generates `.github/workflows/velox.yml`
- `velox init --ci circle` — generates `.circleci/config.yml` (velox job)
- `velox init --ci bitbucket` — generates `bitbucket-pipelines.yml` (velox step)

Each generated workflow:
1. Pulls `ghcr.io/appboy-io/velox`
2. Mounts the test YAML from the repo
3. Passes `TARGET_URL` as an env var (placeholder for dev to fill in)
4. Fails the pipeline if thresholds are breached

## Example Usage

### YAML with env var interpolation

```yaml
name: "API Performance Test"
baseUrl: "${ENV.TARGET_URL}"
flow:
  - name: "Health Check"
    method: GET
    path: "/health"
    threshold:
      p95: "500ms"
```

### GitHub Actions (generated by `velox init --ci github`)

```yaml
name: Performance Test
on:
  workflow_dispatch:
  workflow_call:

jobs:
  perf-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Velox
        run: |
          docker run --rm \
            -v ${{ github.workspace }}/tests:/tests \
            -e TARGET_URL=${{ vars.TARGET_URL }} \
            ghcr.io/appboy-io/velox run /tests/perf.yaml
```

### CircleCI (generated by `velox init --ci circle`)

```yaml
jobs:
  perf-test:
    docker:
      - image: ghcr.io/appboy-io/velox
    steps:
      - checkout
      - run: velox run tests/perf.yaml
```

### Bitbucket Pipelines (generated by `velox init --ci bitbucket`)

```yaml
pipelines:
  custom:
    perf-test:
      - step:
          image: ghcr.io/appboy-io/velox
          script:
            - velox run tests/perf.yaml
```
